# 环境监测系统前端

## 项目简介

环境监测系统前端是基于uni-app框架开发的跨平台应用，用于监测和展示环境数据，包括温度、湿度、空气质量、PM2.5等指标。系统支持多种角色访问，不同角色具有不同的功能权限。

## 角色与权限

系统支持三种角色，每个角色具有不同的功能权限：

### 1. 环保部门

- **实时监测**：城区空气主要污染物自动采集
- **数据分析与预测**：应用大数据与机器学习方法，对各项污染物浓度变化趋势、季节性及空间分布做深入分析
- **设备集成**：支持对喷雾降尘、空气净化等措施的联动自动触发，提升污染治理协同操作效率
- **RAG知识库助手**：RAG知识库集成国内外环保经验，为每次异常警报自动推荐治理与改善措施

### 2. 城市管理者

- **异常预警**：当任一指标超过设定阈值时，自动推送短信/APP/邮件，显示预警级别和建议措施，支持事件跟踪和溯源
- **习惯学习**：系统对历史污染记录分析后，主动优化监测频率与告警触发机制，实现自适应服务
- **数据可视化**：提供热力图、线性趋势、分布雷达等多种可视化方式，一屏掌控整体与局部空气质量状况

### 3. 公众

- **信息公开**：面向社会开放当前及历史空气质量数据，支持按行政区域、时间段搜索，提供健康提示

## 登录与注册

系统提供了完整的登录和注册功能：

1. **登录页面**：用户可以输入账号、密码，并选择角色进行登录
2. **注册页面**：新用户可以创建账号，填写用户名、邮箱、密码，并选择角色
3. **角色切换**：已登录用户可以在首页通过用户菜单切换角色，以访问不同的功能

## 权限控制

系统采用基于角色的访问控制（RBAC）模式：

1. **路由拦截**：未登录用户会被重定向到登录页面
2. **页面权限**：根据用户角色控制可访问的页面
3. **功能权限**：根据用户角色显示/隐藏功能模块
4. **API权限**：后端API根据用户角色和权限进行鉴权

## 主要页面

1. **登录/注册页**：用户认证
2. **首页**：显示环境数据概览和角色对应的功能入口
3. **数据看板**：实时监测数据展示
4. **传感器管理**：设备集成和管理
5. **数据分析**：数据分析与预测功能
6. **区域对比**：不同区域环境数据对比
7. **历史回溯**：历史环境数据查询
8. **异常预警**：预警设置和记录
9. **习惯学习**：系统自适应优化
10. **RAG知识库**：环保知识库和问答系统

## 技术栈

- **前端框架**：uni-app
- **UI组件**：uni-ui
- **图表库**：ECharts
- **状态管理**：Vue Composition API
- **网络请求**：uni.request / axios
- **数据存储**：uni.storage

## 开发与部署

### 开发环境

1. 安装HBuilderX
2. 导入项目
3. 运行项目

### 生产部署

1. 使用HBuilderX发行为H5/App/小程序等平台
2. 配置生产环境API地址
3. 部署到对应平台
